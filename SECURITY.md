# 安全政策

> 繁體中文 | [English](./SECURITY.en.md)

BigGo 購物幫手執行於使用者的瀏覽器中，會與電商網站互動，處理諸如登入狀態、回饋金
session、購物頁面瀏覽等資料。我們重視安全回報，並感謝負責任的揭露。

## 支援版本

安全修正僅套用於各瀏覽器商店上架的**最新版本**。回報前請先確認你使用的是最新版。

| 版本 | 是否支援 |
| ---- | -------- |
| 最新版（Chrome Web Store / AMO） | ✅ |
| 較舊版本 | ❌ |

## 如何回報漏洞

**請勿透過公開的 GitHub issue、discussion 或 pull request 回報安全漏洞。**

請改用以下**其中一種**方式私下回報：

- **GitHub Security Advisories** — 在 repository 的 **Security** 分頁點擊
  「Report a vulnerability」（建議優先使用）。
- **Email** — security@biggo.vip

請盡量包含：

- 漏洞描述與潛在影響。
- 重現步驟（受影響的站台/nindex、瀏覽器與版本、擴充功能版本）。
- 任何 proof-of-concept、log 或截圖。
- 此問題是否已公開，或已為第三方所知。

## 你可以預期

- **3 個工作天內**回覆確認收到。
- **7 個工作天內**完成初步評估（嚴重程度、受影響元件）。
- 我們會持續向你更新修復進度，並與你協調揭露時程。高嚴重度問題我們會盡快修復。
- 在你同意下，我們樂於在修復發佈後給予署名致謝。

## 範圍

**屬於範圍內：**

- 本 repository 中的擴充功能原始碼（background、content script、Svelte frontend、建置工具）。
- 影響擴充功能所處理之使用者資料的機密性、完整性或可用性的漏洞。

**不屬於範圍：**

- BigGo 的後端／API 服務（`*.biggo.com`）——可回報至相同窗口，但其受本公司網站的
  揭露計畫規範。
- 需要裝置已被入侵、實體接觸，或對使用者社交工程才能達成的問題。
- 未實際展示可利用影響的自動化掃描器報告。

## Safe Harbor（免責保障）

對於符合以下條件的研究者，我們不會提起或支持任何法律行動：

- 善意地遵循本政策，
- 避免侵犯隱私、破壞資料與中斷服務，且
- 在任何公開揭露前，給予我們合理的修復時間。

謝謝你協助保護 BigGo 使用者的安全。
